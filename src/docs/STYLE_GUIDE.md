# KeKal Studio Style Guide

## Purpose

This document defines the visual and structural standards used across the KeKal Studio website.

The goal is to ensure consistency across all pages, components, layouts, and future content.

---

# Typography

## Page Title (H1)

Used for:

- Page heroes
- Main page titles

Guidelines:

- One H1 per page
- Largest heading size
- Clear and concise

Examples:

- KeKal Studio
- About KeKal Studio
- Fashion Beyond the Studio

---

## Section Title

Used for:

- SectionHeader component

Examples:

- Featured Collections
- Community Impact
- Collection Story
- Related Products

---

## Card Title

Used for:

- Product cards
- Event cards
- Collection cards

Should remain short and scannable.

---

## Body Text

Used for:

- Paragraphs
- Introductions
- Descriptions

Guidelines:

- Comfortable reading width
- Consistent spacing between paragraphs
- Justified only where it improves readability

---

## Small Text

Used for:

- Dates
- Locations
- Categories
- Metadata

---

# Layout Width

## Standard Container

All standard content should align to a shared content container.

Guidelines:

* max-width: 1200px
* margin: 0 auto
* horizontal padding on smaller screens
* maintain consistent alignment across all pages

Used for:

* Text content
* Product grids
* Collection grids
* Story sections
* Contact sections
* Detail page content

Goal:

Create comfortable reading width and prevent content from sitting too close to screen edges.

---

## Full Width Sections

Only major visual sections may extend beyond the standard container.

Used for:

* Page heroes
* Feature banners
* Large visual storytelling sections
* Upcoming event banners

Goal:

Create visual impact while preserving a consistent content layout elsewhere.

---

# Image Layout Rules

## Full Width Images

Reserved for:

* Hero sections
* Major promotional banners
* Large visual feature sections

Guidelines:

* Image spans full viewport width
* Content may remain aligned to the standard container
* Used intentionally and sparingly

---

## Contained Images

Used for:

* Product content
* Collection content
* Designer content
* Event content
* Community content
* Galleries

Guidelines:

* Image remains inside the standard container
* Aligns with surrounding content
* Preserves page rhythm and consistency

---

# Content Alignment

## Horizontal Alignment

All standard content should align to the same grid.

Examples:

* Section titles
* Body text
* Cards
* Buttons
* Forms

Avoid creating custom widths for individual sections unless there is a strong design reason.

---

## Reading Width

Long-form content should use a comfortable reading width.

Guidelines:

* Avoid extremely wide paragraphs
* Prefer readable line lengths
* Maintain generous whitespace

Goal:

Improve readability and create a calmer visual experience.

## Core Principle

Content is contained.
Impact is full width.

Most sections should respect the standard container.
Only hero sections and major visual banners should break out of it.

## Responsive Layout

The website is built mobile first.

Mobile:
- 16px side padding

Tablet:
- 24px side padding

Desktop:
- 32px side padding
- Content may use a maximum width when needed for readability

Layouts should remain fluid and responsive across all screen sizes.

# Responsive Layout

The website is built mobile first.

Layouts should adapt naturally across screen sizes while maintaining consistent spacing and alignment.

---

## Mobile

Default layout target.

Guidelines:

* Side padding: 16px
* Single-column layouts where appropriate
* Content should use available width comfortably

---

## Tablet

Guidelines:

* Side padding: 24px
* Layouts may expand into wider grids when appropriate
* Maintain generous whitespace

---

## Desktop

Guidelines:

* Side padding: 32px
* Content may use a maximum width for readability
* Avoid excessively wide text blocks

---

# Content Width

Content should remain comfortable to read.

Guidelines:

* Use fluid responsive layouts
* Limit overly wide content areas
* Center content when maximum widths are reached

Goal:

Prevent content from feeling stretched across large screens while preserving responsiveness.

---

# Content Alignment

All major sections should align to the same responsive grid.

This includes:

* Header
* Footer
* Section content
* Forms
* Grids
* Detail page content

Users should perceive a consistent vertical alignment throughout the website.

---

# Spacing Scale

Mobile:

* 16px horizontal spacing

Tablet:

* 24px horizontal spacing

Desktop:

* 32px horizontal spacing

These values should be the foundation for layout spacing across the website.
